Sección de preguntas - ¿Cuál es la diferencia entre Internet y la World Wide Web?
R= internet se refiere a la infraestructura de interconexión de computadoras global, world wide web se refiere a la plataforma para publicar documentos accesibles a través de internet
- ¿Cuáles son las partes de un URL?
R= la primera parte indica protocolo, puede ser http o https. La segunda parte es el nombre asignado por un servicio de hosting, la tercera es el dominio (.com, .edu, .net) y finalmente el subdirectorio.
- ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
R= GET: tiene uso como recuperación de datos
HEAD: similar a GET, pero solo recupera el header de la respuesta sin el cuerpo
POST: envía al servidor un paquete de información
PUT: reemplaza información con el contenido enviado
PATCH: indica modificaciones a un recurso
DELETE: borra un recurso en específico
- ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?
R= POST, debido a que debe encriptarse en el paquete
- ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?
R= GET
- Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?
R= Código 200 indica solicitud exitosa, no se refiere a errores
- ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?
Si es su responsabilidad, debido a que el enclace con el servidor es correcto, pero el recurso solicitado no está disponible
- ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?
R= El error 500 hace referencia a errores del servidor, solo debe resolverlo si es responsable tanto del servidor como de la página web
- ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.
R= Significa que en HTML5 actualmente existe un método que funciona mejor para el propósito requerido, aun no están obsoletos. Pero se recomienda evitar su uso, algunos ejemplos son BASEFONT, FONT y FRONT
- ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?
R= las principales diferencias son la mejor integración de multimedia con HTML5 y la mejora en ciertos métodos para tener un código más limpio en HTML5, además de mejor integración de SQL y JavaScript
- ¿Qué componentes de estructura y estilo tiene una tabla?
R= alineación, color, ancho, numero de columnas, borde, marco, espacio entre columnas, espacio entre filas
- ¿Cuáles son los principales controles de una forma HTML5?
R= input, label, select, textarea, button, fieldset, legend, datalist, output, option, optgroup
